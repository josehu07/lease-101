# Design and Tracker Docs

Use this `docs/` folder for design docs and project plan/tracker docs. Keep all the docs up-to-date with the rest of the codebase at all times.

## Structure

* `DESIGN.md` contains the high-level design overview and plans
* `design/` folder contains narrow-scoped docs, each focusing on one particular aspect of the project
  * Design docs always reflect the ideal state currently in mind; it may be ahead of the code
* `references/` folder contains stashed external references, such as relevant papers
